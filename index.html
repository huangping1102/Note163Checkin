功能特色

强大的自动化签到解决方案

自动化签到

基于 GitHub Actions 的定时任务，每日自动执行签到流程，无需手动操作

多账号支持

支持配置多个有道云笔记账号，批量执行签到任务，提高效率

数据持久化

支持 Redis 缓存和 MySQL 数据库存储，记录签到历史和状态信息

智能通知

集成 Server 酱推送服务，支持微信通知，及时了解签到状态

存储奖励

每日可获得 30M+ 存储空间奖励，通过看广告等方式最大化收益

安全可靠

基于官方 API 接口，安全可靠，支持 Cookie 复用避免频繁登录

安装指南

简单几步，快速部署自动签到服务

1

Fork 仓库

使用 GitHub 账户登录后，点击右上角的 Fork 按钮将仓库复制到您的账户中

2

添加 Secret 配置

在仓库设置中添加名为 CONF 的 Secret，包含账号密码等配置信息

3

运行 Action

在 Actions 页面手动运行工作流，或等待定时任务自动执行

4

查看运行结果

在 Actions 页面查看执行日志，确认签到是否成功

配置说明

详细的配置参数说明

CONF 配置示例

{
  "Users": [
    {
      "Task": "CC",                    // 自定义任务名称，选填
      "Username": "abc@163.com",       // 有道云笔记账号
      "Password": "aaa"                // 账号密码
    },
    {
      "Task": "CC_1",
      "Username": "abcd@163.com", 
      "Password": "bbb"
    }
  ],
  "ScKey": "",                         // Server酱 ScKey，选填
  "ScType": "",                        // 通知类型：Always/Failed，选填
  "RdsServer": "",                     // Redis 服务器地址，选填
  "RdsPwd": "",                        // Redis 密码，选填
  "MySqlServer": "",                   // MySQL 数据库地址，选填
  "MysqlUserName": "",                 // 数据库用户名，选填
  "MySqlPwd": "",                      // 数据库密码，选填
  "MySqlDatabase": ""                  // 数据库名称，选填
}

必填参数

Username/Password

有道云笔记账号和密码

可选参数

ScKey/ScType

Server酱推送配置

Redis/MySQL

数据存储配置

重要提示

  • • 支持配置多个账号同时签到
  • • 建议配置 Redis 以避免频繁登录
  • • 24小时内频繁登录可能触发验证码
  • • 定时任务设置为每天 9:30 和 14:30 执行
